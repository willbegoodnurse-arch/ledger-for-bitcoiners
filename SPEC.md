# My Ledger — 제품 정의 & 기능 명세 (SPEC)

> 이 문서는 Claude Code 등에 넘겨 구현을 의뢰할 때의 **단일 기준 문서**입니다.
> 디자인 레퍼런스 소스: `Lightning Strike.html`, `app.jsx`, `myledger.jsx`, `myledger.css`, `styles.css`, `assets/lightning.png`

## 1. 한 줄 정의
비트코인을 모으는 한국 사용자를 위한 **모바일 가계부 앱**.
원화 가계부에 BTC·사토시 환산, 실시간 시세·김프, 보유 자산 평가까지 결합해
"내 소비와 자산을 사토시 단위로 인식"하게 만든다. — *Stay humble, stack sats.*

## 2. 목적
- 일반 가계부 + **비트코이너 정체성** 결합: 모든 수입·지출·잔액·자산을 ₩와 ₿(사토시)로 동시 표기
- "이 커피 = 몇 사토시"를 체감시켜 **불필요한 소비를 줄이고 적립(stacking)을 독려**
- 시세·김프·보유 평가액을 한 곳에서 확인해 **DCA(분할매수) 의사결정** 지원

## 3. 확정 사항
| 항목 | 결정 |
|---|---|
| 1차 타깃 | **한국 사용자** (업비트·김프 중심) |
| 데이터 입력 | **수기 입력** (API 연동 없음) |
| 추적 범위 | 가계부(현금흐름) **+ 보유 BTC 평가액 / 포트폴리오** |
| 사토시 표기 기준 | **현재 시세로 재평가** (과거 고정 아님) |
| MVP 범위 | 거래 입력 · 통계 · 자산 · 설정 **전부** |
| 플랫폼 | **모바일 앱** |

> 시세 데이터는 수기 입력이 원칙이나, BTC 현재가/환율은 외부 시세를 읽어와 "현재 시세 재평가"에 사용한다(거래 금액·내역 자체는 수기). 시세 소스 연동 범위는 구현 단계에서 결정.

## 4. 화면(탭) 구성 — MVP

### 4.1 홈 / 대시보드 (현재 디자인된 화면)
- 슬로건 `STAY HUMBLE / STACK SATS` (Audiowide, 흰색, 글로우)
- 우측 상단 **번개 효과** (가장 뒤 레이어, 본문 텍스트 위에 안 겹침)
- 헤더: 월 · 앱 이름 · **현재 블록 높이** (펄스 도트)
- **₩ ↔ ₿ 통화 토글** → 전 화면 숫자 동시 변환
- 사용 가능 잔액 (+ 사토시 환산)
- 수입 / 지출 듀얼 카드
- **Bitcoin Price 위젯**: 업비트(KRW) · 바이낸스(USD) · USD/KRW · **김프 %**
- **차트**: 일별 지출(막대) vs BTC 시세(라인), 일/월/년 토글
- 최근 거래 리스트 (카테고리 아이콘 + 현재 시세 기준 사토시 환산)

### 4.2 거래 입력
- 수기 입력 폼: 금액(₩) · 수입/지출 구분 · 카테고리 · 메모 · 날짜/시간
- 입력 즉시 **현재 BTC 시세로 사토시 환산값 미리보기**
- 카테고리: 식비 / 반려 / 투자(BTC) / 수입 / 교통 / 기타 (확장 가능)

### 4.3 통계
- 기간별(일/월/년) 지출·수입 집계
- 카테고리별 분해 (도넛/바)
- 지출 추이 vs BTC 시세 비교
- "이번 달 지출을 사토시로 환산하면 N sats" 인사이트

### 4.4 자산 / 포트폴리오
- 보유 BTC 수량 · **현재 평가액(₩)** · 평가손익
- 평균 매입가 대비 수익률
- 적립 추이(사토시 누적) 그래프
- ※ 현재 시세로 항상 재평가

### 4.5 설정
- 기본 통화(₩/₿) · 표시 단위(BTC/sats)
- 시세 소스 / 새로고침 주기
- 카테고리 관리 · 테마(다크 고정) · 백업/내보내기

## 5. 디자인 시스템 (요약 — 상세는 CSS 파일 참조)
- 다크 테마 고정. 배경 `#0a0a0a`, 반투명 카드 + `backdrop-blur(10px)`
- 액센트 **Bitcoin Orange `#F7931A`** (1~2곳만 강하게, 본문 텍스트엔 미사용)
- 수입/긍정 `#4ade80`, 지출/부정 `#f87171`
- 폰트: 본문 `Pretendard Variable` / 데이터(블록·사토시) `mono` / 슬로건 `Audiowide`
- 큰 숫자 `tabular-nums`, 라벨 `uppercase` + tracking
- 번개 오버레이: z-index 0(최하단) + 본문 하단 검정 그라디언트로 가독성 보장

## 6. 포맷 규칙
- `fmtKRW` → `₩1,847,500` (음수 `-₩…`)
- `fmtBTC(krw, 현재시세)` → `≈ 0.01169 BTC`, 0.001 미만은 `≈ 1,169,303 sats`
- 사토시 환산은 **항상 현재 시세** 기준 (저장은 ₩ 원본 + 환산은 표시 시 계산)
- 김프 = `(업비트KRW − 바이낸스USD × USDKRW) / (바이낸스USD × USDKRW) × 100`

## 7. 비범위 (MVP 제외)
- 거래소/은행 **API 자동 연동** (수기 입력만)
- 다중 계정/공유 가계부
- 알트코인 / 멀티 자산 (BTC 단일)
- 라이트닝 결제 연동(브랜딩 모티프일 뿐 실제 결제 아님)

## 8. 기술 스택 제안
- 모바일: **React Native (Expo)** 권장 — 단, 디자인 레퍼런스는 React 웹으로 제작됨.
  웹 프로토타입을 그대로 받아 RN으로 포팅하거나, 웹앱(PWA)으로 먼저 검증 후 RN 이행.
- 상태: 로컬 우선(수기 입력 데이터는 온디바이스 저장)
- 차트: 경량 SVG 직접 구현
